# Feature Specification: sharebox MVP

**Feature Branch**: `001-sharebox-mvp`
**Created**: 2026-04-12
**Status**: Draft
**Input**: User description: "sharebox MVP — 個人用マルチデバイス共有Webアプリ。自分が所有する複数デバイス間でテキスト・ファイル・URLを素早く安全にやり取りする個人専用ツール。Google OAuth ホワイトリスト認証で本人以外のアクセスを拒否し、統合タイムライン上で3種類の投稿を時系列管理する。"

## User Scenarios & Testing _(mandatory)_

### User Story 1 - Authenticated text sharing across devices (Priority: P1)

開発者本人が、自分の Google アカウントで sharebox にログインし、任意のテキスト
(URL、メモ、コード片、パスフレーズ等)を PC から投稿し、別のデバイス(スマホ)
から同じテキストを確認してクリップボードへコピーできる。不要になった投稿は自分で
削除できる。

**Why this priority**: 本アプリの中核体験であり、これ単体で既存の「Slack 自分用
DM」「iCloud 共有メモ」の代替として成立する最小のユニットである。認証とタイムライン
という全機能共通の土台もこのストーリーで完成するため、他ストーリーの前提となる。

**Independent Test**: 開発者本人の Google アカウントでログイン後、PC から文字列
を投稿し、スマホで同アプリを開いて同じ文字列が新しい順に表示され、コピー操作
によって OS のクリップボードに正しく入ること、および投稿を削除するとタイムライン
から消えることを一連の操作で確認できる。

**Acceptance Scenarios**:

1. **Given** 未ログイン状態でアプリにアクセスした、**When** Google OAuth でホワイト
   リスト登録済みメールでログインした、**Then** タイムライン画面に遷移する。
2. **Given** ホワイトリスト外のメールアドレスでログインを試みた、**When** Google 認証
   が完了した、**Then** アクセス拒否メッセージが表示されタイムラインには遷移しない。
3. **Given** ログイン済みでタイムラインを開いている、**When** テキスト入力欄に文字列
   を入力して投稿ボタンを押した、**Then** 投稿がタイムライン先頭に表示される。
4. **Given** PC で投稿したテキストが存在する、**When** 同一アカウントでスマホから
   ログインしてタイムラインを開いた、**Then** 同じテキストが時系列順に表示される。
5. **Given** テキスト投稿が表示されている、**When** 該当投稿のクリップボードアイコン
   をタップした、**Then** OS のクリップボードにテキストが格納され、コピー成功の
   視覚フィードバックが表示される。
6. **Given** 自分が投稿したテキストが存在する、**When** 削除操作を実行した、
   **Then** 投稿はタイムラインから消え、再読み込みしても復活しない。
7. **Given** ログイン中である、**When** ログアウト操作を実行した、**Then** セッション
   が破棄されログイン画面にリダイレクトされる。

### User Story 2 - File sharing with previews and download (Priority: P2)

開発者本人が、スマホで撮影した画像・動画・PDF・その他ファイルをアップロードし、
別デバイスからプレビューを確認したうえでダウンロードできる。不要になったファイル
は投稿とともにストレージからも削除できる。

**Why this priority**: ファイル共有は利用頻度の高い機能だが、テキスト共有
(US1)がなくても独立して動作可能であると同時に、US1 がない状態ではアプリ全体
として魅力が弱いため、P1 の後に積む P2 とする。

**Independent Test**: 認証済み状態でファイルをアップロードし、タイムラインに
プレビュー/アイコンとメタデータ(ファイル名・サイズ)が表示され、別デバイス
からダウンロードリンクで完全なファイルが取得でき、削除操作後は一覧からもストレージ
からも消えていることを確認できる。

**Acceptance Scenarios**:

1. **Given** タイムライン画面を開いている、**When** ファイルをドラッグ&ドロップ
   または選択ダイアログで選んでアップロードした、**Then** アップロード中に進捗が
   表示され、完了後タイムラインに該当エントリが追加される。
2. **Given** 画像または動画をアップロードした、**When** タイムラインに表示された、
   **Then** サムネイルまたはインラインプレビューが表示される。
3. **Given** 画像・動画以外のファイル(PDF、zip など)をアップロードした、**When**
   タイムラインに表示された、**Then** ファイル形式に対応したアイコン、ファイル名、
   ファイルサイズが表示される。
4. **Given** タイムラインにファイルエントリが表示されている、**When** 「ダウンロード」
   リンクをクリック/タップした、**Then** 別デバイスからでもファイルの実体が正しく
   ダウンロードされる。
5. **Given** 50MB を超えるファイルをアップロードしようとした、**When** アップロード
   操作を実行した、**Then** 「サイズ上限 50MB を超えています」等の明確なエラー
   メッセージが表示され、アップロードは拒否される。
6. **Given** 自分がアップロードしたファイルが存在する、**When** 削除操作を実行した、
   **Then** タイムラインからエントリが消え、ストレージからも実体が削除される。

### User Story 3 - URL sharing with OGP preview cards (Priority: P3)

開発者本人が、閲覧中のWebページのURLを sharebox に投稿し、OGP(og:title /
og:description / og:image)を取得してカード形式で表示する。別デバイスから
そのカードをタップして元ページを開いたり、URL 文字列をクリップボードにコピーしたり
できる。

**Why this priority**: 既存ブラウザのブックマーク機能やメッセンジャーへの「リンク
送信」で代替可能なため、P1/P2 より優先度は下がるが、Notion風のOGPカードUIは
本アプリの体験価値を底上げするため MVP スコープ内に含める。

**Independent Test**: 認証済み状態で既知のWebページURL(例: 主要なニュース
サイトやGitHubリポジトリ)を投稿し、タイムラインにOGPカード(タイトル・説明・
サムネイル画像)が表示されること、カードクリックで元ページが新しいタブで開く
こと、コピーアイコンで URL がクリップボードに入ること、削除が機能することを
確認できる。

**Acceptance Scenarios**:

1. **Given** タイムライン画面を開いている、**When** URL 入力欄に有効な URL を
   貼り付けて投稿した、**Then** タイムラインに該当エントリが追加され、OGP取得
   中のプレースホルダ表示の後、OGPカード(タイトル・説明・画像)に差し替わる。
2. **Given** OGP を取得できないURL(OGメタが無い、サーバーが応答しない等)
   を投稿した、**When** タイムラインに表示された、**Then** フォールバック表示
   (URL文字列とドメイン名のみ)が表示される。
3. **Given** タイムラインにURLエントリが表示されている、**When** カード本体
   またはURLテキストをクリック/タップした、**Then** 新しいタブ/ウインドウで
   該当ページが開かれる。
4. **Given** URLエントリが表示されている、**When** クリップボードアイコンを
   クリック/タップした、**Then** URL文字列がクリップボードに格納され、成功
   フィードバックが表示される。
5. **Given** 自分が投稿した URL が存在する、**When** 削除操作を実行した、
   **Then** タイムラインからエントリが消え、再読み込みしても復活しない。

### Edge Cases

- **アップロード中のネットワーク切断**: ファイルアップロード中に通信が途切れた
  場合、ユーザーに失敗を通知し、途中状態のエントリをタイムラインに残さない。
- **同一URLの重複投稿**: 同じ URL を複数回投稿することは許可する(時系列メモ
  として扱う)。
- **空のテキスト投稿**: 空白のみのテキストは投稿できないようにし、エラーを
  示す。
- **極端に長いテキスト**: 単一のテキスト投稿に上限(100,000 文字相当)を
  設け、超過時は明確にエラー表示する。
- **OGP取得のタイムアウト**: OGP 取得は 5 秒以内に応答しない場合フォールバック
  表示に切り替える。
- **未対応プロトコルのURL**: `http(s)` 以外のスキーム(`ftp://` 等)は投稿を
  拒否する。
- **タイムラインが空の状態**: エントリが1件もない場合、空状態を示す案内
  (「最初の投稿をしてみましょう」等)を表示する。
- **同時編集/削除の競合**: 複数デバイスで同じエントリを同時に操作した場合、
  最後に到達した操作が優先され、矛盾した UI 状態にならないこと。
- **セッション失効**: サーバー側でセッションが失効した状態で操作した場合、
  自動的にログイン画面に戻し、操作途中のデータを失わないよう配慮する。
- **ホワイトリスト外アカウントでのログイン試行**: 明示的に拒否し、ログを残す
  (不正ログインの痕跡を本人があとで確認できるよう)。

## Requirements _(mandatory)_

### Functional Requirements

#### 認証・アクセス制御

- **FR-001**: System MUST 認証手段として Google OAuth を提供し、他の認証手段
  (メール/パスワード等)は提供しない。
- **FR-002**: System MUST ホワイトリストに登録された特定の単一メールアドレス
  のみログインを許可し、それ以外のメールアドレスのログインを拒否する。
- **FR-003**: System MUST ログインページおよび OAuth コールバック以外のすべての
  ページ・API エンドポイントに対し、未認証アクセスを拒否しログインページへ
  リダイレクトする。
- **FR-004**: Users MUST be able to 明示的なログアウト操作でセッションを破棄できる。
- **FR-005**: System MUST 拒否されたログイン試行(ホワイトリスト外、OAuth 失敗)
  を本人が後から確認できる形で記録する。

#### 投稿の共通機能

- **FR-006**: System MUST テキスト・ファイル・URL の 3 種類の投稿を単一の
  統合タイムライン上で時系列(新しい順)に表示する。
- **FR-007**: System MUST 各エントリに投稿日時を相対表示(例: "3 分前", "昨日",
  "3 日前")で表示する。
- **FR-008**: Users MUST be able to 投稿種別(全て / テキストのみ / ファイルのみ /
  URL のみ)でタイムラインをフィルタリングできる。
- **FR-009**: Users MUST be able to 自分のすべての投稿を削除できる。削除は即時反映
  され、タイムラインから消え、再読み込み後も復活しない。
- **FR-010**: System MUST 投稿はユーザーが明示的に削除するまで永続保存する。
  自動削除・自動アーカイブは行わない。
- **FR-011**: System MUST 他デバイスからタイムラインを開いたとき、最新の投稿
  が反映された状態を取得できる(明示的な再読み込みを含む)。

#### テキスト共有

- **FR-012**: Users MUST be able to テキスト入力欄に文字列を入力・貼り付けし、
  投稿操作でタイムラインに追加できる。
- **FR-013**: System MUST 空白のみのテキスト投稿を拒否する。
- **FR-014**: System MUST 1 投稿あたりのテキスト長に上限 (100,000 文字) を設け、
  超過時は明確なエラーメッセージを表示する。
- **FR-015**: Users MUST be able to 各テキストエントリ横のクリップボードアイコン
  を操作することで、OS のクリップボードにテキストをコピーし、コピー成功の視覚
  フィードバックを受けられる。

#### ファイル共有

- **FR-016**: Users MUST be able to ドラッグ&ドロップおよびファイル選択ダイアログ
  からファイルをアップロードできる。
- **FR-017**: System MUST 1 ファイルあたりのサイズ上限を 50MB とし、超過ファイル
  のアップロードを拒否して明確なエラーメッセージを表示する。
- **FR-018**: System MUST アップロード中に進捗(パーセンテージまたはプログレス
  バー)を表示する。
- **FR-019**: System MUST 画像ファイルにはサムネイルまたはインラインプレビュー
  を表示する。
- **FR-020**: System MUST 動画ファイルには再生可能なインラインプレビューまたは
  サムネイルを表示する(初期ロードをブロックしない遅延読み込みを行う)。
- **FR-021**: System MUST 画像・動画以外のファイル(PDF、文書、圧縮ファイル等)
  には、ファイル形式に対応したアイコン・ファイル名・ファイルサイズを表示する。
- **FR-022**: Users MUST be able to 各ファイルエントリの「ダウンロード」リンク
  から、別デバイスで元のファイル実体をダウンロードできる。
- **FR-023**: System MUST ファイル削除時に、タイムラインエントリだけでなく
  ストレージ上のファイル実体も削除する。
- **FR-024**: System MUST アップロード中のネットワーク切断・エラー時に、中途
  半端なエントリをタイムラインに残さず、ユーザーに失敗を通知する。

#### URL 共有

- **FR-025**: Users MUST be able to URL 入力欄に文字列を貼り付け、投稿操作で
  タイムラインに追加できる。
- **FR-026**: System MUST 投稿された URL のスキームが `http(s)` 以外の場合、
  投稿を拒否する。
- **FR-027**: System MUST 投稿された URL の OGP (og:title, og:description,
  og:image) を取得し、カード形式でタイムラインに表示する。
- **FR-028**: System MUST OGP 取得に失敗した場合(タイムアウト、メタタグ不在、
  取得エラー等)、URL 文字列とドメイン名のみのフォールバック表示を行う。
- **FR-029**: System MUST OGP 取得にタイムアウト(5 秒以内)を設け、超過時は
  フォールバック表示に切り替える。
- **FR-030**: Users MUST be able to URL カード全体または URL テキスト部分を
  クリック/タップすることで、新しいタブ/ウインドウで該当ページを開ける。
- **FR-031**: Users MUST be able to 各 URL エントリ横のクリップボードアイコン
  を操作することで、URL 文字列をクリップボードにコピーできる。

#### 非機能要件

- **FR-032**: System MUST レスポンシブ UI を提供し、モバイル(<600px)・タブレット
  (600-1080px)・デスクトップ(>1080px)で同等の主要機能を利用できる。
- **FR-033**: System MUST すべての画面で DESIGN.md に記載されたデザインシステム
  (カラーパレット、タイポグラフィ、余白、影、角丸)に準拠する。
- **FR-034**: _(撤回)_ — 旧: 総ストレージ使用量を本人に可視化する。個人用 MVP に
  不要と判断し撤回。番号は履歴のため保持し、FR の renumber は行わない。
- **FR-035**: System MUST Core Web Vitals の "Good" 閾値(LCP < 2.5s,
  INP < 200ms, CLS < 0.1)をすべての主要画面で満たす。

### Key Entities

- **User (Owner)**: アプリ唯一の利用者。Google OAuth プロフィール(メールアドレス、
  表示名、アバター画像 URL 等)と、ホワイトリストに登録された一意のメール
  アドレスを持つ。セッションを通じて認証状態を保持する。
- **Share Entry**: タイムラインに表示される投稿の共通抽象概念。種別(テキスト/
  ファイル/URL)、作成日時、作成者(常に Owner)、種別ごとの本体データへの
  参照を持つ。削除操作の対象となる単位。
- **Text Post**: Share Entry の一種。テキスト本文を保持する。
- **File Post**: Share Entry の一種。元ファイル名、ファイルサイズ(バイト)、
  MIME タイプ、ストレージ上の実体への参照、必要に応じて生成済みプレビュー情報
  を保持する。
- **URL Post**: Share Entry の一種。元 URL 文字列と、取得済みの OGP 情報
  (タイトル、説明、画像 URL、取得ステータス、取得日時)を保持する。
- **Login Audit Record**: ログイン試行の記録。試行日時、試行したメールアドレス、
  結果(成功 / ホワイトリスト拒否 / OAuth 失敗)、発信元 IP 相当の情報を保持
  する。本人が後から不正アクセスの痕跡を確認するために使用する。

## Success Criteria _(mandatory)_

### Measurable Outcomes

- **SC-001**: 認証済みユーザーは、PC で投稿したテキストを **5 秒以内** に
  スマホ側のタイムラインで確認でき、コピー操作で即座にクリップボードへ反映
  できる(ユーザーが感じる体感速度として)。
- **SC-002**: 50MB 以内のファイルアップロードが、通常のモバイル通信環境で
  **30 秒以内** に完了する(上限付近のファイルで)。
- **SC-003**: URL 投稿から OGP カード表示までの所要時間が **3 秒以内** である
  (OGP 取得可能な代表的なページで)。
- **SC-004**: すべての主要画面(ログイン、タイムライン、投稿、プレビュー)が
  Core Web Vitals の "Good" 閾値(LCP < 2.5s, INP < 200ms, CLS < 0.1)を満たす。
- **SC-005**: ホワイトリスト外の Google アカウントでのログイン試行は
  **100% 拒否** され、タイムラインや API にアクセスできない。
- **SC-006**: US1〜US3 の主要ユーザーフローが自動化 E2E テストで完全に再現され、
  継続的インテグレーション環境で **すべて成功** する状態で本番デプロイされる。
- **SC-007**: 本番運用コストが **月額 0 円** で維持され、無料枠を超過しない。
- **SC-008**: 本番環境に無料サブドメインでデプロイされ、本人のスマホ・PC 双方
  から問題なくアクセスできる。
- **SC-009**: _(撤回)_ — 旧: 総ストレージ使用量を 80% 到達前に可視化する。
  FR-034 の撤回に伴い削除。番号は履歴のため保持。

## Assumptions

- 唯一のユーザーは開発者本人のみであり、将来にわたって他ユーザー・他人への共有
  機能は追加しない。そのため、権限管理・共有リンク発行・レート制御などの
  マルチテナント機能は設計対象外である。
- Google アカウントが本人によって安全に管理されている(Google 側の 2 段階認証
  等のアカウント保護は本アプリの責任範囲外)。
- すべてのデバイスは最新の主要ブラウザ(Chrome / Safari / Edge / Firefox)を
  利用し、クリップボード API・ドラッグ&ドロップ API・OAuth2 ポップアップ等の
  モダン Web API が利用可能である。
- 本人のデバイスはインターネット接続が原則あり、オフライン動作・オフライン
  投稿キューは本 MVP のスコープ外である。
- データの自動バックアップ・エクスポート・長期アーカイブは本 MVP のスコープ外
  (永続保存を前提とするが、耐障害性の責任はインフラ提供元の SLA に準じる)。
- デザインシステムの単一の真実源は `DESIGN.md` であり、新規トークン追加時は
  実装に先立って同ファイルを更新する運用とする。
- 「ゼロコスト運用」の制約から、ファイルダウンロード時のエグレス料金が発生
  しないインフラを前提とする。
- フォルダ・タグ・横断検索・コメント・編集機能・PWA 化は v1.0 MVP のスコープ外
  とし、将来の v2 以降で必要性が生じた時点で再評価する。
